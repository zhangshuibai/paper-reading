TL;DR — 这篇博客的主线是一句话：想让 test-time compute 真正换成 accuracy，verifier / reward 信号必须回流到训练里——纯 training-free 的 TTS 在渐近意义上是次优的。 它以 Aviral Kumar 组 2025–2026 的工作为骨架，串起一条清晰的演进：① 先证明 training-free 撞墙（$\Omega(\sqrt{H})$ 的差距、以及无监督 RLVR 必然「先涨后崩」）；② 把 TTS 写进训练目标（MRT 的 progress reward）；③ 从「想更多」升级到「探索对」（e3 让 in-context exploration 成为可训练技能、实现算力外推）；④ 啃难题（Privileged On-Policy Exploration 用 oracle 前缀把 on-policy rollout 拉进高 reward 区）；⑤ 给 long-CoT 做内部 credit assignment（InT 自提单步干预）；⑥ agent 设定里改成 scale「交互次数」而非「思考长度」（TTI）。最后落到一个判断：不必执着 training-free，工业界也不需要；真正的蓝海是把 TTS 与 On-Policy Distillation / Continual Learning 接起来。

全景路线图

整篇文章可以读成一条「把 test-time 这件事，一步步搬进训练目标」的主线。下图把六个环节串起来：

①起点：training-free 为什么 scale 不动

前两篇博客覆盖了两条 training-free 路线：采样侧（best-of-N、verifier reranking、search）与 in-context 侧（self-refine、self-evolve）。它们的共同点是权重永远不动——拿一个冻结的 base model，往里灌推理时算力去把 accuracy 曲线顶上去。博客的判断是：这条路能 work，直到 work 不动为止，而这堵墙是有名字的。

《Scaling Test-Time Compute Without Verification or RL is Suboptimal》（ICML’25 Spotlight，Kumar 组）给出一个很锋利的结论：在 verifier-based（VB）与 verifier-free（VF）的二分下，存在一族 base policy，使 VB 在算力 scaling 上对 VF 有 $\Omega(\sqrt{H})$ 的渐近优势（$H$ 是算力预算）。直觉上，VF 路线（典型如把 reasoning trace 蒸回 SFT）只是在围着 policy 的均值做 imitation，无法主动放大「稀有但正确」的尾巴；而 VB 把 verifier 信号回流到训练，等价于一次 importance-weighted update，天然能利用 reward 的 anti-concentration 把那条尾巴「钓」上来。一句话：要让 test-time compute 真换成 accuracy，verifier / reward 必须流回训练。

一个自然的反驳是：verifier 信号能不能「免费」自举出来？比如 TTRL 那条线——多数投票伪标签、entropy minimization，本质是拿模型自身的一致性当 reward，不要外部 verifier。这能不能补上 $\Omega(\sqrt{H})$ 的差距？《How Far Can Unsupervised RLVR Scale LLM Training?》（ICLR’26）把这条退路也堵死了：在一个统一框架下，作者指出所有「intrinsic reward」做的其实是同一件事——distribution sharpening，把 policy 指数级地塌缩到一个由模型 prior 决定的不动点。经验上的指纹是一条普适的「先涨后崩」曲线：先有增益，然后在 Model Collapse Step 处掉头向下，而这个拐点的位置由模型自身的 confidence-correctness 对齐程度决定、与你喂多少数据无关。论文的实务结论是：intrinsic reward 只在小集合（<128 样本）上安全——拿来做 test-time training 没问题，但不能替代真正的 verifier 去做大规模 post-training。

这一节的立论 training-free 与「无监督自奖励」都受同一条规律约束：它们只能锐化模型已有的分布，不能注入新的、被外部信号验证过的能力。所以下文所有方法的共同动作，都是把某种被验证的信号塞进训练目标。

②把 TTS 写进训练目标：MRT

《Optimizing Test-Time Compute via Meta Reinforcement Fine-Tuning》（MRT，ICML’25 Spotlight）从一个观察出发：标准的 outcome-reward RL 只在一条 trace 的最末尾发一个 0/1，于是模型分不清「我在第 500 个 token 就做对了」和「我到第 5000 个 token 才上道」。结果是一条抖动的 scaling 曲线——加预算并不可靠地有用。

MRT 把目标改写成 meta-RL 式的累积 regret（cumulative regret）：把一条 reasoning trace 切成若干 chunk，给每个 chunk 一个 progress reward $\Delta P(\text{success})$——即「经过这个 chunk 之后，success 概率的边际增量」。于是模型同时被两件事推着走：结尾要对，并且沿途要单调地越来越对。论文报告了 2–3 倍的相对算力效率提升。

博客在这里点了题：一旦你接受「训练目标里必须含 test-time」这个前提，后面所有工作都可以读成在这根主干上、朝不同方向打的补丁。

③从「想更多」到「探索对」：e3

MRT 把目标和 test-time 曲线对齐了，但还有一个正交的失败模式：长 CoT ≠ 真正的 exploration。模型可以再多吐五千个 token 的 CoT，而 accuracy 指针纹丝不动。《e3: Learning to Explore Enables Extrapolation of Test-Time Compute》（ICLR’26）把「in-context exploration」提升为一项头等的、可训练的技能。e3 的配方有三味药：

  • Asymmetries（非对称性）。 base model 通常验证比生成更强，于是鼓励 trace 内部的操作：self-verification、self-refutation、局部 backtracking。
  • Negative-gradient reinforcement（负梯度强化）。 在失败的 trace 上主动放大 exploration 分支，让模型敢于分叉。
  • 结构化探索 + 耦合 curriculum。 沿着由浅入深的结构化 exploration 模式来训练。

结果是第一次可靠地实现了 test-time compute 的外推：训练时只见过 16k token 的模型，在测试时 32k＋ token 仍能持续涨 accuracy，外推因子约 2×。

④探索难题：Privileged On-Policy Exploration

e3 解决的是「愿意探索」。难题还有一个「你够不够得到 reward」的问题：难题上的 on-policy reward 基本全是 0，梯度立不起来；而在 easy/hard 混合集上做 RL 又会撞上「ray interference」——优化方向被 easy 题劫持，hard 题永远学不会。

《Learning to Reason on Hard Problems with Privileged On-Policy Exploration》（arXiv 2601.18779）玩了个巧妙的把戏：把一段 oracle solution 作为privileged prefix（特权前缀）拼在难题前面，让模型在它的条件下做 on-policy rollout。注意这不是 off-policy distillation——rollout 仍然是 on-policy 的，只是 conditioning 分布被 oracle 拉进了高 reward 区域。论文报告：约 10% 此前无解的难题变得可解，pass@1 / pass@k 跳到 58% / 83%。

⑤long-CoT 的内部 credit assignment：InT

训练里一大批失败的 trace 只贡献一个 0 信号，极其浪费。《InT: Self-Proposed Interventions Enable Credit Assignment in LLM Reasoning》（ICLR’26）让模型读自己失败的 trace，定位第一个出错的 step，并提出一个单步 intervention（最小的反事实编辑）。参考解只用来验证那一步——验一个编辑很便宜，从头生成一条完整正确 trace 很贵：又一处可套利的 asymmetry。

被验证通过的 intervention 在 likelihood 上被上调，这就自动产出了一个稠密的 step-level credit-assignment 信号。把它当作下游 RL 的 initializer，比纯 RL 大约高 10 分。博客的类比很到位：可以把 InT 读成 MRT progress reward 的离散表亲——MRT 给连续的 $\Delta P$，InT 给单步的「这个 token 本该是那个 token」的证据。

⑥Agent 设定不同：用「交互」scaling 取代「思考」scaling

《Thinking vs. Doing: Agents that Reason by Scaling Test-Time Interactions》（TTI，NeurIPS’25 Best Paper）抛出一个让 agent 社区有点扎心的论点：在 agentic 任务里，想更多没用，做更多才有用。一旦环境本身会返回信息，再往里堆内心独白 token 的边际收益就快速衰减。test-time scaling 该选的轴是交互次数（number of interactions），而不是思考长度。

TTI 用一套 curriculum-online RL：逐步增大 rollout horizon $h$，让模型学会自适应地决定「再探一步，还是收尾」。博客顺势呼应了它此前在《Agentic RL: Decoupling Reasoning and Tool-use》里的主张——reasoning 的 token 预算与 tool-use 的 step 预算是两件不同的事，必须分开 scale、分开训练。

⑦Takeaway 与尚未定论的争议

RL 到底有没有教给模型新东西，仍有争议。一种偏学术的观点认为 RL 本质上是拿 pass@k 换 pass@1，只是让模型锐化自己的分布（Power Sampling，ICLR’26 / Scalable Power Sampling，ICML’26）。而 entropy、logits、self-certainty 这些 intrinsic 指标，是否真和模型能力或 RL 有效性相关，也远未定论——既不能说强相关，也不能说不相关。

一种工业视角则认为：学术界研究的 RL 模型（Qwen3）太弱了，只有换到 gpt-oss-120b 这种工业真正在用的模型上，问题才会有结论。博客认为 gpt-oss-120b 是个货真价实的好 RL 模型、堪称典范的 post-RL 模型，是「弱模型 / 强模型」的分界线：它的 generation length、entropy、self-certainty 都表现得合理得多——一个标定良好的模型，output length 与 accuracy 之间应当呈干净的单调递减关系，而学术规模的 Qwen3 实验永远到不了那里。

博客的落点 Test-time scaling 本质就是用推理时的 exploration 抬升模型表现；而无监督 TTS——哪怕它能套进 RLHF 目标（ETS，ICML’26）——仍逃不出那条 suboptimality。结论是务实的：我们不必执着于完全 training-free，工业界也不需要 training-free。如何从学术角度把 TTS 更好地对齐工业需求，是来年一个又长又蓝海的方向。再叠加当下 On-Policy Distillation（OPD）的爆发——SDFT、SDPO 这类工作正在用 OPD 改写旧的 RL 范式，大家都在朝 Continual Learning 漂移——没人知道今天这套 RL 还能走几年。

作者列出的未来方向

  • Continual Learning & Self-Evolving：面向 long-horizon 的 LLM / VLM / Agent reasoning。
  • 新的 RL 前沿：On-Policy Distillation（OPD）——SDPO、SDFT。
  • 大小模型协作：Speculative Decoding、MTP。
