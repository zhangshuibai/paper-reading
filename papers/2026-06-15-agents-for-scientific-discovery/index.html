一句话 — agent 正把 AI 从「AI for Science（当工具：预测、模拟）」推向「Agentic Science（当科研主体：自己提假设、设计实验、跑实验、分析、迭代）」。整个科研闭环——文献综述 → 提想法 → 设计实验 → 执行 → 分析 → 写论文——每一环都已有代表系统：dry-lab 端到端的 AI Scientist / Agent Laboratory、做假设的 Google AI co-scientist、连机器人跑湿实验的 Coscientist / ChemCrow / A-Lab、生物医药的 BioDiscoveryAgent / Virtual Biotech。但「真做出新科学」与「看起来像在做科学」仍有距离：novelty/rigor、可复现性、wet-lab 落地、评测标准都还不成熟。

全景：科学发现的「闭环」与自主性阶梯

多篇 2025–2026 综述（如《From AI for Science to Agentic Science》arXiv 2508.14111）都把科研抽象成一个可迭代的闭环，并按 agent 的自主程度分级：从「assistant（人主导、AI 打下手）」→「collaborator（人机协作）」→「autonomous agent（AI 自己跑完闭环）」。下图是这个闭环（也对应「文献→想法→实验→分析→写作」六阶段工作流）：

实践里通常把系统分两类：dry-lab（纯计算/写作闭环，不碰真实仪器）与 wet-lab / self-driving lab（接机器人真做实验）。下面按应用方向展开。

①端到端「AI Scientist」研究 agent（dry-lab 全流程）

把「提想法 → 写代码做实验 → 分析 → 写成论文」整条流水线自动化，多为多 agent 分工（hypothesis generator / experimenter / analyst / writer），主要在计算型学科（ML、数据科学）上跑得动。

系统干什么 / 亮点出处
The AI Scientist（Sakana AI）全自动从 idea→实验→写出完整 manuscript；v2 自动生成的 3 篇投 ICLR workshop、1 篇被接收（里程碑，但 novelty 争议大）2408.06292
Agent Laboratory把 LLM agent 当研究助理，走 literature review → experimentation → report，强调人可在每阶段介入2501.04227
AutoSOTA / URSA / OmniScientist面向「自动刷 SOTA」「通用科研 agent」「人-AI 科学家共演化生态」的更新一代系统2604.05550 · 2506.22653 · 2511.16931
AI Scientist 类的批判性评测有独立评测指出当前「AI Scientist」更接近「看起来像研究」，离 Artificial Research Intelligence 尚远2502.14297

和本库其它解读的联系这条线和库里的 SIA（自我改进 agent）、MAI-Thinking-1（把研发当系统优化的 hill-climbing machine）同源——都是「让 AI 参与/自动化研发本身」。AI Scientist 也是 SIA 论文引用的相关工作之一。

②假设生成与文献发现（hypothesis generation）

不追求全自动跑实验，而是聚焦科研最「值钱」的一步——提出新颖、可验证的假设，再交给人类去验证。代表是 Google / DeepMind 的 AI co-scientist。

系统机制 / 亮点出处
Google AI co-scientistGemini 2.0 驱动的多 agent 系统：self-play 式科学辩论生成假设、ranking 锦标赛比较假设、evolution 迭代提质；定位「虚拟科研合作者」，已报道在药物重定位、抗菌素耐药性机制等生物医学假设上给出可被实验验证的结果Google Research · DeepMind
literature-based discovery / idea agents从海量文献里挖「未被连接的知识」生成假设、做 novelty/significance 评估（多见于 AI4Research 综述）2507.01903
方法论可迁移点 AI co-scientist 的「生成 → 辩论 → 锦标赛排序 → 进化」其实就是把test-time scaling / 自我博弈用在假设空间上——和本库里 TTRL（多数投票自评）、SIA（自改进闭环）是同一套「让模型自己生成-评判-迭代」的思想，只是对象从答案换成了科学假设。

③化学 / 材料 + 自驱实验室（wet-lab：闭环到机器人）

这是 agent「真动手做实验」最成熟的方向——LLM 当大脑，调工具/写代码控制机器人跑合成与表征，形成 self-driving lab（SDL） 闭环（常被归纳为 Comprehension → Design → Execution → Analysis → Optimization 五模块）。

④生物 / 生物医学 / 药物发现

数据极度异构（基因组、转录组、影像、临床…），agent 多用于自动化分析、实验设计、靶点发现，并向「多 agent 虚拟药企」演进。

⑤怎么评测科学发现 agent

「会写论文 ≠ 会做科学」，所以评测是这领域的关键基础设施。主流 benchmark 分「数据驱动任务正确性」「真实分析可复现」「虚拟实验室探索」几类。

Benchmark考什么出处
ScienceAgentBench（ICLR'25）102 个数据驱动发现任务，取自 44 篇同行评审论文，覆盖 bioinformatics / 计算化学 / GIS / 心理与认知神经，9 位专家验证2410.05080
BixBench计算生物/生信，真实分析「capsule」（基因组/转录组/差异表达/系统发育…）2503.00096
CORE-Bench聚焦可复现性：给齐材料与环境，看 agent 能否复现已发表的计算分析2409.11363
DiscoveryWorld虚拟实验室：在模拟仪器里让 agent 跨生物/化学/物理做完整发现循环2406.06769
LAB-Bench / LAB-Bench2（FutureHouse）生物研究实操能力（文献、图表、协议、克隆…）2407.10362 · 2604.09554

⑥现状判断与挑战

三条主线 ① 自主性在分层推进。计算型学科（ML/数据科学）已能跑「准自动」的端到端闭环；化学/材料靠机器人做到了 wet-lab 闭环；生物医药多停在「分析+实验设计助手」+ 多 agent 协作雏形。

② 最稳的价值不是「全自动出论文」，而是「假设/实验设计的加速器」。AI co-scientist、BioDiscoveryAgent 这类「人来拍板、AI 扩搜索空间」的协作范式，比「一键生成论文」更经得起检验。

③ 方法上和「自我改进 / test-time scaling」收敛。生成-辩论-排序-进化（co-scientist）、闭环假设迭代（BioDiscoveryAgent）、主动学习优化（A-Lab/ARROWS）本质都是「让系统自己生成-评判-迭代」。
公认的坑 novelty / rigor 存疑（Sakana AI Scientist 的「新颖性」被多方质疑）；可复现性差、易hallucinate 引用与结论；wet-lab 落地仍是少数明星案例、难规模化；评测不统一（多为计算任务正确性，难度量「真发现」）；以及安全与治理（自主实验的生物/化学风险）。这也是为什么 Mozi 强调「governed autonomy」、co-scientist 强调「human-in-the-loop」。

给想入场者的实务建议

  • 选对自主层级：先做「协作式假设/分析加速器」，比硬上「全自动科学家」更易出真价值、也更易评测。
  • 闭环要有可验证 reward：化学/材料有真机表征、计算学科有可执行验证——优先选「结果能被客观判分」的子领域（呼应 ScienceAgentBench/CORE-Bench 的可验证设计）。
  • 诚实评测：用 ScienceAgentBench / BixBench / CORE-Bench 这类有 ground-truth 的基准，别只看「生成的论文像不像」。
