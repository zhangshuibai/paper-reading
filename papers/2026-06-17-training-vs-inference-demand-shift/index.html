训练 vs 推理：算力转向如何重排 AI 供应链需求（含 red-team 修订）

当算力重心从「训练为主」漂向「推理为主」，AI 供应链哪些位置需求飙升、哪些降温、哪些公司更关键——并把「初版直觉结论 → red-team 修正」的对照摆在正中央。

来源等级：🟢 一手 官方/规范/论文 · 🟡 二手 媒体/分析 · 🔴 自述/有强反条件的情景； 份额/产能/份额转移数字一律「约 / 数量级 / 视年份」并标代表公司，不编精确值。

⚠ 图里列出的所有公司：非投资建议 / 非背书，仅标其在产业链中的位置；多数为有强反条件的情景判断，受益≠确定。

一句话 — 推理不是均质负载（prefill 算力密集 / decode 显存带宽密集 / reasoning 长思维链把两者一起放大），算力转向把需求重心从「峰值 FLOPS + 最大同步 scale-out」漂向「显存带宽与容量 + 每 token 成本/每瓦 + scale-up 域内带宽 + 分离式 KV 搬运」；但本篇的灵魂是 red-team 修订——最危险的初版直觉是「推理＝只吃显存、网络降权」，实际上 reasoning/分离式推理把 compute 与 scale-up 网络一起拉回来；真正零替代的卡点更可能是 CoWoS/先进封装 而非 HBM，而受益高度集中在 Broadcom/Marvell/TSMC，并非「广义第三方供应链普涨」。

①领域背景速补（讲给硬件小白）

本篇假设你懂 LLM、但不熟 AI 硬件供应链。先把后面反复用到的几个硬件名词讲清——尤其要打破「推理就是显卡跑一下、比训练简单」的直觉，因为推理根本不是一种均质负载。

推理的三种状态：prefill / decode / reasoning

一次 LLM 推理至少分成性格迥异的两段，再加上长思维链这个放大器：

  • prefill（预填充，处理输入）🟢：把整段 prompt 一次性喂进模型、并行算出所有输入 token 的中间表示。这是稠密矩阵乘，瓶颈是算力（compute-bound）——拼的是峰值 FLOPS。类比：把一本厚书一口气读完做笔记，越快越好，吃的是「脑子转速」。
  • decode（解码，逐 token 生成）🟢：自回归地一个 token 一个 token 往外吐。每生成一个 token 都要把模型权重和缓存从显存搬一遍，算的却很少，于是瓶颈是显存带宽（bandwidth-bound），而不是算力。类比：一个字一个字地写回信，手不累，但要不停翻查资料，吃的是「翻书速度」（带宽）。
  • reasoning（长思维链 / o1 式推理）🟡：模型在给出答案前先生成大量「思考」token。它把 prefill（要反复读越来越长的上下文）和 decode（要生成海量中间 token）同时放大，于是compute 回潮——纠正了「推理一定比训练省算力」的旧印象。这是本篇最关键的背景：reasoning-heavy 的推理对 compute / memory / network 三样一起拉。

KV cache：容量瓶颈的来源

KV cache（键值缓存）🟢是推理时缓存下来的中间结果（每个已处理 token 的 Key/Value 向量），这样 decode 时不必把整段上下文重算一遍。它的麻烦在于大小随「上下文长度 × 并发数」线性膨胀——上下文越长、同时服务的用户越多，KV cache 就越大，很快吃光显存容量。所以它是容量瓶颈（capacity）的主要来源，和「带宽不够」是两个不同的墙。

带宽瓶颈 vs 容量瓶颈（要求不同供应商）

同样是「显存（HBM）不够」，可以是两种完全不同的不够：带宽不够（搬得不够快，decode 卡这里）要的是更高代际、更快的 HBM；容量不够（装不下 KV cache）要的是更多 GB、更便宜的每 GB，甚至可以把 KV 卸载到更慢但更大的存储（NAND/CXL 内存）。这两条墙对应不同的受益方——这正是后面「HBM＝最确定受益」需要被 red-team 拆开的原因：带宽和容量不分，结论就会走样。

scale-up vs scale-out（两种「把卡连起来」）

  • scale-up（纵向扩展，机柜内）🟢：把一个机柜内的几十张 GPU 用NVLink 域（高带宽、低延迟、近距离铜/光）连成一个「超级大芯片」，让它们像共享显存一样协作。这是「域内带宽」。
  • scale-out（横向扩展，跨机柜）🟢：把很多机柜用以太网/InfiniBand + 光模块跨距离连起来，规模可达数万卡。训练时的「最大同步全互联」主要压在这里。

初版最大的硬伤就是把「训练式超大 scale-out 全互联软化」错当成「网络整体降权」——而推理（尤其分离式）恰恰在加权 scale-up 域内带宽。

分离式推理 disaggregation

分离式推理（prefill/decode disaggregation）🟢把算力密集的 prefill 和带宽密集的 decode 拆到不同机器上，各自配最合适的硬件（prefill 堆 FLOPS、decode 堆 HBM 带宽），从而都跑在最划算的点上。代价：prefill 算好的 KV cache 要跨机搬运到 decode 节点，于是凭空多出一条对网络（NVLink / NVLink-C2C / RDMA，如 NVIDIA Dynamo + NIXL）的新需求。🟡vLLM、SGLang、TensorRT-LLM、NVIDIA Dynamo 等在 2026 已普遍支持，DeepSeek 等在生产中大规模部署。这是「推理拉动网络」的直接机制，也是纠正「推理＝网络降权」的关键证据。

⑥给小白的心智模型

把整条 AI 供应链想成一座高速公路收费系统：

  • 训练像「一次性铺一条全新八车道高速」——爆发式 capex，要在同一时刻把最多的车（GPU）用最宽的同步立交（超大 scale-out 全互联）连起来，峰值功率拉满。
  • 推理像「这条路天天有车流跑、要持续收费运营」——opex 长期化，关心的是每公里油耗（每 token 成本/每瓦）、收费站吞吐（带宽）和停车场容量（KV cache 容量）。
  • 初版直觉以为「运营阶段只要加油站（HBM）就行，立交桥（网络）可以拆」。red-team 提醒：reasoning 让车流既跑得远（compute）、又频繁进出停车场（带宽/容量）、还要在分仓运营（分离式推理）之间来回运货（KV 跨机搬运）——于是机柜内的高架快速路（scale-up）反而更挤，立交不能拆，只是从「跨城超级立交」换成「园区内高架」。
  • 而真正卡脖子的不是加油站，是「能造收费道闸的唯一工厂」（CoWoS/先进封装）——加油站有三家（HBM），道闸只有一家（TSMC CoWoS），谁更有定价权一目了然。

⑦依赖与相互制约（深入）

核心因果链：推理转向 ≠ 第三方供应链普涨

初版的隐含假设是「需求漂移＝把蛋糕换个位置切，大家都能分到」。red-team 拆出两条对广义第三方链的下行路径：

  1. 份额转移是零和：当推理转向定制 ASIC，NVIDIA 的单芯片份额↓，但接住的不是「ASIC 生态」，而是Broadcom / Marvell / TSMC 旱涝保收（它们做代设计/制造，不管谁赢都收钱）；独立 ASIC（Groq/Cerebras）可能是输家。所以「ASIC 利好」对绝大多数第三方其实是集中度上升，不是普惠。
  2. 两条结构性下行：(a) 超大厂自研集中化——hyperscaler 把芯片/网络/电源越来越多收进自研体系，挤压通用第三方器件；(b) 端侧分流——越多推理跑在手机/PC（Qualcomm/Apple/Arm/联发科/AMD APU），云推理链的边际需求越被抽走。两条都指向「广义第三方链未必随推理量同步涨」。

单源 vs 竞争，决定卡点的定价权

同样在「需求拉动」下，能不能把需求变成利润取决于替代性：CoWoS（近单源 TSMC）＞ ASIC 代设计（Broadcom/Marvell 双寡头）+ TSMC ＞ HBM（三家竞争）。HBM 需求再旺，三家互卷 + 量化降带宽 + 扩产，就可能把「最确定受益」变成「2027 过剩尾险」；而 CoWoS 一家说了算，需求 3× 供给，定价权完全不同。这就是为什么修订把先进封装上调、把 HBM 从「最确定」降级。

相互制约因果链（推理侧） reasoning-heavy 推理 →（长思维链 + 长上下文）prefill 算力回潮（撞 compute / FLOPS 墙）→ 同时 decode 吃 HBM 带宽（撞带宽墙）→ KV cache 随上下文×并发膨胀（撞容量墙）→ 为各取所长上分离式推理→ KV cache 必须跨机搬运（撞 scale-up 域内带宽 / NVLink-C2C / RDMA 墙）→ 机柜功率密度上探 1MW（撞芯片级电源/散热墙）→ 这一切的硅都要过CoWoS 先进封装（撞最终的零替代墙）。每一步「想更大/更快」都撞一堵不同供应商的墙——所以推理同时拉动 compute / memory / network，而非单纯 memory。
最易被误植的一环 「网络降权」之所以危险，是因为它把一种网络（训练式超大同步 scale-out）的软化，错当成所有网络降权。而推理新增的两类网络需求——scale-up 域内带宽与分离式 KV 跨机搬运——恰恰在加权，对应 CPO / scale-up 光 / 高速互联。把这两者混为一谈，是初版方向性最可能翻车的地方。
