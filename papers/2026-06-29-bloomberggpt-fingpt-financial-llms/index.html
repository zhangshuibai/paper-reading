BloombergGPT vs FinGPT：金融大模型的两条路线（综述）

一边花几百万美元从头训一个专用金融大模型并锁起来，一边主张「别从头训」——用几百美元 LoRA 微调开源模型 + 数据为中心。两条路线之争，外加一个更扎心的现实判断。

一句话 — 金融大模型早期走出两条相反的路：BloombergGPT（2023，约 50B 参数）选择烧钱从头训一个闭源专用模型——在 Bloomberg 私有金融语料上训练、模型不发布；FinGPT（2023，AI4Finance）则主张「别从头训」——用 LoRA 微调开源 LLM + RAG + 数据为中心，强调便宜（自述单次微调约几百美元 🔴）、可复现、能频繁更新（金融数据时效性极强）。但今天最重要的判断是：通用前沿模型（GPT-4 / Claude 等）在很多金融任务上已追平甚至超过专用金融 LLM，让「是否值得做专用金融大模型」本身受质疑；前沿正从「单体金融大模型」转向 RAG over 金融数据 + 金融 agent（如 TradingAgents）。专用金融 LLM 真正的护城河在私有金融数据，而非模型本身。

①两条路线之争

2023 年，「能不能给金融做一个专属大模型」这个问题，几乎同时收到了两种针锋相对的答卷。

  • 重资产派（BloombergGPT）：金融语言有它自己的「方言」——SEC filing（上市公司向美国证监会提交的法定披露文件，如 10-K 年报）、财报电话会、行情术语、彭博终端的私有数据。Bloomberg 的赌注是：把这些私有语料攒起来，从零训练一个 50B 的专用模型，让它在金融 NLP 任务上碾压通用模型。代价是几百万美元的算力和一个不对外发布的闭源模型。
  • 轻资产派（FinGPT）：AI4Finance 的反问是——金融数据天天在变（今天的新闻明天就过期），你从头训一次要烧几百万、还训得慢，怎么跟得上市场？ 不如站在开源 LLM 的肩膀上，用 LoRA 这种参数高效微调（只训极少量新增参数）+ RAG（检索增强，让模型即时查最新资料）快速贴合金融，把功夫下在数据管线和频繁更新上，而不是模型权重本身。这就是它的「data-centric / 数据为中心」哲学。
一句话区分 BloombergGPT 的核心资产是「私有数据 + 从头训出来的那一个闭源大模型」；FinGPT 的核心资产是「开源 + 一条便宜、可复现、能不断重训的数据-微调流水线」。前者卖「成品」，后者卖「持续生产成品的方法」。

②BloombergGPT：闭源、从头训的专用金融大模型

是什么：Bloomberg 与 Johns Hopkins 大学合作的金融大语言模型，论文 arXiv 2303.17564（2023，Wu、Irsoy 等）。它是首个公开报告的大规模专用金融 LLM，被广泛当作这条路线的标杆。

  • 参数规模：约 50B（500 亿） 🟢。论文与官方稿一致；架构为 BLOOM 风格的 decoder-only，70 层、隐藏维 7510 🟡。50B 是按其算力预算下的「Chinchilla 最优」规模选的。
  • 训练数据：约 7000 亿（700B）token 量级 🟢。其中 Bloomberg 自建的金融语料 FinPile 约 363B token（金融新闻、公司披露 / SEC filing、新闻稿、网页、彭博专有数据），再混入约 345B token 的通用语料（约一半金融、一半通用）；论文报告实际训练消耗约 569B token。
  • 训练方式：从头预训练（from scratch） 🟢——不是在别人模型上微调，而是从随机初始化开始训。
  • 算力 / 成本：约 130 万 A100 GPU 小时（512 张 40GB A100、约 53 天）🟢。换算成钱常被引为「约 $300 万 / 次」量级 🟡（成本是按云价折算的估计，非官方账单，口径会随 GPU 单价波动）。
  • 开源情况：闭源，模型权重未发布 🟢。论文公开了方法与训练日志（"chronicles"），但不放模型——这正是 FinGPT 后来主打的反差点。

强在哪：论文报告它在金融 NLP 任务上整体优于同规模通用模型，同时不牺牲通用能力。典型金融任务包括：sentiment（情绪分析，判断一段财经文本看涨/看跌/中性）、NER（命名实体识别，从文本里抽出公司/人名/金额等实体）、金融问答与分类。

意义 BloombergGPT 的历史地位是「证明了专用金融大模型这件事能做、且在金融 NLP 上确有增益」的第一块里程碑。但它把模型锁在内部，意味着外部研究者无法复现、无法在其上继续建设——这恰好给开源路线留出了叙事空间。

③FinGPT：开源、廉价微调、数据为中心

是什么：AI4Finance Foundation（Xiao-Yang Liu 等）的开源金融 LLM 项目，论文 arXiv 2306.06031（2023），代码与模型在 GitHub: AI4Finance-Foundation/FinGPT、权重在 HuggingFace。

哲学（核心）：不从头训。 FinGPT 明确把自己定位成 BloombergGPT 的反面——既然金融数据时效性强、从头训又贵又慢，就复用预训练好的开源 LLM，只做轻量适配：

  • LoRA / QLoRA 微调 🟢：Low-Rank Adaptation 只训练新增的低秩矩阵，冻结原模型绝大部分权重。论文一例中 LoRA 引入的可训练参数仅约 8.3M，不到 8B 基座模型的 0.1%，因此显存占用和训练时间都大幅下降。
  • 数据为中心（data-centric）+ 自动标注 🟢：FinGPT 强调真正的活儿在数据。一个聪明点子是用「市场自带的标签」——把一条新闻发布后对应股价的相对涨跌幅，当作该新闻情绪的监督标签，省去人工标注。
  • RAG（检索增强） 🟢：FinGPT-RAG 在情绪分析里接入外部知识检索，让模型回答时即时拿到最新上下文，缓解「模型权重里的知识会过期」的问题。
  • 便宜 + 可频繁更新（卖点）：项目自述单次微调成本可低至约 $300 量级 🔴，并据此宣称「约为从头训 BloombergGPT 的万分之一」🔴——这是 FinGPT 方的对比说法，便宜到可以每月甚至每周重训以跟上市场，是它相对闭源大模型最实在的差异化。

组件 / 应用（据 GitHub）：

  • 数据管线：data source 层 + 实时 NLP 数据工程层，覆盖多市场新闻 / 行情。
  • 模型 / 任务：多版 FinGPT-Sentiment-Analysis（v1 市场标签 / v2 LLM 标签 / v3 学术数据集），演示 LoRA 的效果。
  • FinGPT-Forecaster 🟢：一个「初级 robo-advisor（机器人投顾，自动给投资建议的程序）」demo，输入一家公司，输出综合分析 + 下周股价走势预测；有 HuggingFace Space 演示，发布过 fingpt-forecaster_dow30_llama2-7b_lora（在 DOW30 上、2022-12 至 2023-09 数据上微调）。
  • FinRobot：AI4Finance 后续推出的开源金融 agent 平台（更偏「用 LLM 搭金融分析 agent」，是 FinGPT 路线的自然延伸）。
需谨慎对待的自述 「FinGPT 在某些任务上超过 BloombergGPT」是 FinGPT 方的报告口径 🔴，且两者评测集 / 提示词 / 基座未必完全可比——把它当「开源廉价方案也能打」的存在性证据看，而不是「全面碾压」的定论。同理「$300/次」「万分之一成本」是其宣传对比，量级可信，但精确数字依基座、硬件、数据量浮动。

⑤更大的图景：金融 LLM landscape 与一个扎心判断

把两条路线放回整个金融 NLP/LLM 谱系，会看到一条清晰的演进线，以及一个让「专用金融大模型」叙事松动的现实。

谱系：从 FinBERT 到金融 agent

  • FinBERT（早期） 🟡：BERT 时代的金融情绪分析模型，证明「预训练语言模型用在金融文本上能显著提升」，是这条线的起点。
  • PIXIU / FinMA + FLARE benchmark（arXiv 2306.05443）🟢：开源的「金融 LLM + 指令数据 + 评测基准」三件套。FinMA 是在 136K 金融指令数据（FIT）上微调 LLaMA 得到的开源金融模型；FLARE 是配套 benchmark（5 类金融 NLP 任务 + 1 类预测任务），让金融 LLM 第一次有了相对统一的「考卷」。其报告 FinMA 在若干任务上分数高于更大的通用模型甚至 BloombergGPT 🔴（同样是论文自述口径）。
  • FinTral（arXiv 2402.10986）🟡：基于 Mistral-7B 的多模态金融模型（文本 + 数字 + 表格 + 图），自述全面超过 ChatGPT-3.5、部分任务超 GPT-4——代表「金融 LLM 也走向多模态 + 对标前沿」的方向。
  • 金融推理 / agent 模型：最新方向不再追求「单体大模型」，而是把 LLM 当组件去搭系统。代表是 TradingAgents（arXiv 2412.20138，GitHub）🟢：模仿交易公司分工，用多个 LLM agent 扮演基本面/情绪/技术分析师、看多看空研究员、风控、交易员，互相辩论后决策。
最重要的现实判断 通用前沿模型（GPT-4 / Claude 等）在很多金融任务上已经追平甚至超过专用金融 LLM。 这直接动摇了 BloombergGPT 式叙事——当一个没专门为金融训过的通用模型，靠强大的通用推理 + 一点提示工程就能把金融情绪/问答做得很好时，「专门花几百万从头训一个金融大模型」的性价比就成问题了。于是前沿重心明显从「造一个金融大模型」转向「RAG over 金融数据 + 金融 agent」：用通用强模型 + 检索最新私有数据 + 多 agent 协作，而不是把知识硬塞进权重。🟡（landscape 综合判断，非单一权威数字）

⑥现实判断 + 局限

  • BloombergGPT 是里程碑，但已被通用前沿模型部分盖过。 它证明了「专用金融大模型可行且有增益」，但闭源 + 从头训的重资产打法，在通用模型能力暴涨后性价比下降——它更像「一个时代的标杆」而非「当下最优解」。
  • FinGPT 的开源 / 廉价 / 数据为中心更可持续。 金融数据天然要频繁更新，能用几百美元就重训一次的流水线，比一个训一次几百万的庞然大物更契合这个领域的节奏。但要清醒：它的「超过 BloombergGPT」「$300/次」是自述口径，需独立验证。
  • 专用金融 LLM 的护城河在「私有金融数据」，而非模型本身。 既然通用模型 + RAG 就能贴合金融，模型权重越来越不是壁垒；真正稀缺的是 Bloomberg 那种独家、实时、结构化的金融数据。这也解释了为何前沿在往「RAG over 私有数据 + agent」走——价值从「模型」转移到「数据 + 编排」。
  • 评测要警惕 lookahead bias / 数据泄漏。 这是金融 LLM 评测的特有陷阱：在互联网级语料上预训练的模型，可能在训练时就「见过」未来的真实结果（lookahead bias，即用了评测时点之后才该知道的信息）。研究已发现 LLM 能近乎逐字复述其训练截止前的历史经济/股价数值——这意味着许多「LLM 能预测股价」的亮眼回测，可能部分来自记忆而非真预测。缓解手段包括实体/日期脱敏提示、时间感知预训练、按时间戳严格切分等。🟡（A Test of Lookahead Bias in LLM Forecasts 等）
一句话收尾 与其问「BloombergGPT 和 FinGPT 谁对」，不如看到：两条路线都已被「通用强模型 + RAG + 金融 agent」这条新主线超越或吸收。BloombergGPT 留下「专用可行」的证明，FinGPT 留下「便宜可复现可持续」的方法论，而金融 AI 的下一站，是把强通用模型接到独家私有数据和多 agent 工作流上——并时刻提防回测里的数据泄漏。
