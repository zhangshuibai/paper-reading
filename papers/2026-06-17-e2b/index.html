一句话 — E2B（公司主体 FoundryLabs / E2B，e2b.dev）做的是「给 AI agent 跑代码的安全云沙箱」：开源 SDK（Python/JS）+ 托管云 + self-host，底层用 Firecracker microVM，~80–200ms 秒级启动、强隔离，让 LLM 安全执行生成的代码、装包、读写文件、联网——典型用在 code interpreter、data analysis agent、coding agent、computer-use。赛道判 emerging → 快速 crowded：2024–2025 资本与玩家集中涌入（Modal、Daytona、Morph、Runloop、Vercel Sandbox、Fly.io Sprites、Northflank…），且 Cloudflare 2026 已 GA 自家 Sandboxes——巨头进场是拥挤化的强信号。最大威胁不是同行而是上游：大模型厂自带 code execution（OpenAI Code Interpreter、各家内置）+ Cloudflare/Vercel 顺手做。可借鉴 whitespace：面向中国信创/私有化部署的 agent 沙箱（数据不出域、国产 CPU/OS 适配），以及 computer-use / GUI 沙箱这一更高价值的垂直层。

消歧 本篇说的 E2B 是 e2b.dev——AI agent 的代码执行 sandbox / runtime，YC / 捷克裔创始人、开源 SDK，公司法人主体在融资文件里常写作 FoundryLabs, Inc.（品牌 E2B，意为 "environment to business"）。不是其它同名「E2B」（如药物警戒数据标准 ICH E2B(R3)、各类 B2B/电商缩写、生物学名词）。文中如再遇歧义会就地点明。🟡

①赛道地图 + 早期度判断

E2B 属于 「agent infra / 卖铲子」层里的一个细分：code execution sandbox / agent runtime。需求来自一个很硬的现实——LLM / agent 会生成代码并想真的「跑起来」（做数据分析、调试、装依赖、写文件、点浏览器），但你不能把模型生成的、不可信的代码直接丢进自己的生产环境跑。于是需要一个秒级启动、强隔离、用完即弃、可联网可持久化的「一次性电脑」。E2B 把这件事打包成 SDK + 云：sbx = Sandbox() 一行起一个隔离 Linux 环境，跑完销毁。🟢

玩家分群（四象限别漏）

  • 现有巨头 / 上游平台：Cloudflare（2026 年 Sandboxes GA + Dynamic Workers 用 V8 isolate 跑 agent 代码）、Vercel（Vercel Sandbox，beta，Firecracker）、AWS（Firecracker 的原主、Lambda/Fargate 可自建）。这些平台「顺手做」就是 E2B 最危险的方向。🟡
  • 垂直创业公司（直接竞品）：Modal（serverless + Sandboxes，唯一沙箱内带 GPU）、Daytona（主打最快冷启动）、Morph（Infinibranch，VM 分叉/并行探索）、Runloop（Devboxes，面向 coding agent / 评测）、Northflank、Blaxel、Fly.io Sprites（2026 年 1 月发布）、Coder。🟡
  • 大模型厂自带 code execution（最大替代威胁）：OpenAI Code Interpreter / Advanced Data Analysis、各大模型厂（Anthropic、Google 等）内置的 code-tool / agent runtime——对终端开发者而言「闭环自带」往往就够用，根本不需要再外采沙箱。🟡
  • 开源 / 免费替代（最易被忽视）：E2B 自己就是开源的（Apache-2.0），所以「自建」门槛被它亲手拉低——团队可以拿 Firecracker / gVisor / Docker + Kata Containers 自己搭；还有 Replit、CodeSandbox 等通用云 IDE/沙箱可改造。🟡
早期度判断 emerging → 正快速 crowded （nascent 萌芽 / emerging 新兴 / crowded 拥挤 / mature 成熟）

判断信号： ① 玩家数量与新增速度——18 个月内冒出十几家专门做 agent 沙箱的公司，2026 年仍有新玩家（Fly.io Sprites 1 月发布）🟡； ② 融资热度——E2B Series A $21M、Daytona Series A $24M（传 $125M 估值）、Modal Series B $87M（$1.1B 估值，传 2026 年谈 $2.5B），资本明显在押这一层🟡； ③ 巨头是否进场——已进场：Cloudflare 2026 年 Sandboxes GA、Vercel 出 Sandbox、AWS 持有 Firecracker，这是赛道从 emerging 滑向 crowded 的最强信号🟡； ④ 开源 / 免费替代——存在且强（E2B 本身开源 + Firecracker/gVisor 自建）； ⑤ 客户认知——已成型（Manus / Perplexity / Mistral 等知名 agent 都在用沙箱），但「用谁家」远未固化； ⑥ 公认 benchmark / 标准——尚无统一标准，各家自报冷启动/隔离/计费口径打架（见下表注意事项）。

综合：需求真实且爆发（agent / coding agent 起飞 → 执行层刚需），但供给端 18 个月就挤满，且巨头已下场——窗口正在快速关闭。

②竞品对照矩阵

选 7 个对这个赛道最有决定性的维度：开源/闭源（决定自建门槛与信任）、隔离技术（microVM 强隔离 vs 容器/isolate，决定安全与冷启动的权衡）、启动延迟（agent 循环里每步都在等，极敏感）、持久化/状态（pause-resume / snapshot / fork，决定能不能跑长任务）、computer-use 支持（GUI/桌面沙箱，高价值场景）、自托管（决定能不能进信创/私有化）、融资规模（耐力）。各家自报口径不一，已尽量标来源；冷启动数字尤其是营销樱桃挑选，谨慎对待。

读矩阵：E2B 的差异化在三点——开源 + microVM 强隔离 + 自托管三件套同时具备，加上较早把 computer-use / Desktop 沙箱做出来，且采用面最广（Manus/Perplexity/Mistral 等都用）。但同质化已很严重：microVM 路线被 Vercel、Fly.io、Runloop、Morph 复制；冷启动这条「卷点」被 Daytona、Blaxel 卷到几十毫秒；GPU 这条 Modal 独占；钱最多的是 Modal（$1.1B 估值）。E2B 的真正护城河不在某个技术指标，而在开源生态 + 先发的开发者心智 + 企业自托管能力——但这三样都不抗「上游平台顺手做」。

⑦中国对标：agent 代码沙箱在中国可行吗

中国做「agent 代码执行 / 函数计算 / 云端隔离沙箱」的力量，主要不是独立创业公司，而是三朵云顺手把这层做进自己的 serverless / 容器产品，外加各 coding agent 厂自建沙箱：

  • 阿里云（动作最齐）：函数计算 FC 推出 AIO 沙箱（统一云端隔离环境，预装 puppeteer / playwright 做浏览器 + 代码执行）、Sandbox Agent & Skills（公测）、AgentRun（基于 Serverless FC 的 agentic 沙箱）；容器计算服务 ACS 的 Agent Sandbox 直接对标 E2B——号称 MicroVM 级隔离 + 内存级休眠/唤醒 + Checkpoint 克隆 + 每分钟弹 15K 沙箱。这是最像 E2B 的国产路线，且天然绑「百炼」大模型平台。🟡
  • 火山引擎 / 腾讯云 SCF：均有 serverless 函数计算可承载 agent 代码执行，但本次检索未找到与 E2B 同等定位、单独打「agent sandbox」品牌的一手页面——记为未找到明确对标产品（不代表没有）。🔴
  • coding agent 自建沙箱：国内 coding/agent 产品（各家 AI IDE、agent 平台）多自建执行环境，而非外采——这压缩了独立沙箱创业公司的空间。🔴
  • E2B 国产开源替代：未检索到一个广泛采用、专门对标 E2B 的国产开源沙箱项目；团队更多直接基于 Firecracker / gVisor / Kata 自建。🔴
中国可行性 + whitespace 「独立 agent 沙箱公司」在中国可行性偏弱：三朵云（尤其阿里云 ACS Agent Sandbox / FC AIO）已把这层做进自家 PaaS，并绑大模型平台，独立创业者很难在通用沙箱上竞争。但有两块明确 whitespace：① 信创 / 私有化部署——金融、政务、国企要求数据不出域、适配国产 CPU(鲲鹏/海光)与 OS(麒麟/欧拉)，公有云沙箱进不去，这块需要可私有化、可信创认证的 agent 沙箱；② 垂直场景沙箱——把代码沙箱与具体行业工作流（金融研报、数据分析、RPA/computer-use）打包成开箱即用方案，而不是卖底层 runtime。

⑧创业 idea 借鉴

Whitespace / 机会 定位图右上角的复合空位：「可私有化/信创合规」× 「GUI/computer-use 垂直沙箱」。通用代码沙箱已被挤满且巨头下场，但「数据不出域的 agent 沙箱」（信创/私有化）与「专为 computer-use / GUI 自动化设计、带状态可分叉的沙箱」这两格，现有玩家都没专门占——前者是合规护城河，后者是更高价值的执行层。

现有玩家共同短板（unmet needs）

  • 云成本透传、毛利薄：所有沙箱公司本质是在云上加一层，毛利被底层云成本压；缺乏「比自建明显更省」的硬理由时，大客户会自建。
  • 口径混乱、缺标准：冷启动/隔离/计费各家自报、无公认 benchmark，买家难比较——谁先做出可信第三方 benchmark / 兼容标准，谁就能定义这个市场。
  • 合规/私有化是缺口：主流玩家以公有托管为主，受监管行业（金融/政务/医疗）落地难。
  • computer-use 仍粗糙：GUI 沙箱的稳定性、可观测、可回放、可分叉仍不成熟。

Why now（为什么现在值得做）

coding agent / agentic 应用爆发 → 执行层成刚需：agent 从「会说」走向「会做」，每一步「做」都要一个安全可弃的执行环境。叠加 Firecracker/gVisor 等开源隔离技术成熟、秒级 microVM 成为标配、按秒计费的云原生计费模型可行——这是「为什么是 2024–2026」。但窗口正在关闭（巨头已进场），所以 why-now 也意味着「要么垂直、要么合规，别再做一个通用沙箱」。

可借鉴的打法 + 相邻机会

  • borrowable：① 开源 SDK 引流 + 托管云变现的经典 PLG 打法（开源拉开发者心智，云端按用量收钱）；② 用具名旗舰客户 case study（Perplexity 一周上线、Manus 虚拟电脑）做信任背书，比堆 logo 更有说服力；③ 把「难的基础设施（microVM 隔离）封成一行 API」的产品化思路。
  • adjacent：① 信创/私有化 agent 沙箱（中国 whitespace，合规即护城河）；② computer-use / GUI 垂直沙箱（更高价值、更难被模型厂顺手做）；③ 沙箱可观测/评测层（给所有沙箱做 benchmark / trace / 回放，吃「标准缺失」红利，且不与底层云直接拼成本）；④ 垂直行业沙箱（金融数据分析、生信、BI——把 runtime + 行业数据/工具打包）。
风险（诚实给） ① 大模型厂自带 code execution 是最大替代威胁——开发者用闭环自带的就够，外采沙箱的需求被上游吃掉；② Cloudflare / Vercel / AWS 顺手做——已 GA，绑平台分发，价格可补贴；③ 开源自建——E2B 自己开源 + Firecracker 现成，大客户自建门槛低；④ 毛利被云成本压，资本密集，而 E2B 弹药($32–35M)远少于 Modal($111M)；⑤ 赛道 18 个月就拥挤、同质化严重，差异化窗口短。

⑨现实判断

E2B 是这条赛道里定位最清晰、开发者心智最强、采用面最广的独立玩家之一：开源 + microVM + 自托管 + computer-use 的组合在直接竞品里相对完整，Perplexity / Manus 等具名案例是真实采用信号。胜算在于「开源生态 + 企业 self-host」能吃到「不想被锁进 Cloudflare/Vercel、又要合规自托管」的客户。但软肋很硬：① 技术指标无一项绝对领先（冷启动被 Daytona 卷、GPU 被 Modal 占、计费各家趋同）；② 弹药量明显小于 Modal；③ 最大威胁来自上游——模型厂自带执行 + Cloudflare/Vercel 顺手做，二者都能绕过它。赛道判 emerging→快速 crowded：需求真、玩家多、巨头已下场，对新进入者意味着别再做通用沙箱，要么往合规/信创（私有化）走，要么往垂直 computer-use/行业沙箱走。
