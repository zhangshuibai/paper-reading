一句话脉络 — 最朴素的 agentic SFT = 「整段 trajectory ＋ 均匀 cross-entropy ＋ mask 掉环境输出」的行为克隆。最近两年的算法研究都在攻它的三个软肋：① 监督信号太糙（该 mask 谁、该重点学哪一步）；② 数据从哪来、质量如何（合成、拒绝采样、自蒸馏、失败挖掘）；③ 纯模仿的病理（mode collapse、只学到「接口捷径」而非真本事，泛化差），并因此越来越往「与 RL 融合 / cold-start」走。下面按 6 个方向归类代表工作。

①监督信号设计：mask 谁、让什么进 loss

最基础的算法问题：一条轨迹里哪些 token 算 loss。共识是只监督 assistant 生成的部分（reasoning＋tool_call＋答案），mask 掉 system/user/tool-response（否则模型会学着编造环境返回）。近期工作开始质疑这套默认掩码是否最优：

工作类型算法点（一句话）
ToolLLM / ToolBenchSFTReAct 轨迹 SFT 的范式奠基：loss 只在 Thought＋Action，Observation 被 mask
Chat-template assistant mask（HF / TRL assistant_only_loss、LLaMA-Factory）SFT机制层：{% generation %} 标记 assistant token；gpt/function 学、observation mask
ACC（Agent Context Compilation）SFT反向操作：把轨迹「编译」成 long-context QA，让答案梯度绕过 mask、触达证据 token，补上长程监督盲区
What Do Agents Learn from Trajectory-SFT: Semantics or Interfaces?SFT警示：trajectory-SFT 会放大「接口捷径（interface shortcutting）」——接口一改就崩，说明学到的是表面格式而非任务语义

相关ECHO（terminal agent）则把「预测环境输出」当作 auxiliary loss（与 mask 相反），用作 world-model 式的稠密自监督——见同库 ECHO 解读。

②多轮组样本与训练效率

一条 K 轮会话怎么变成训练样本？普通多轮可整段一条（mask 同时学所有轮）；带 thinking 的多轮因历史思考要从后续 context 删掉，被迫按轮拆成 N 条、跑 N 次 forward（$O(N^3)$）。算法创新主要在「不拆也等价」：

工作类型算法点
One-Pass to ReasonSFTtoken 复制 ＋ block-sparse attention mask，在单次 forward 得到与 N-pass 完全相同的 loss，$O(N^3)\to O(N^2)$
Qwen3 官方多轮建议SFT带 reasoning 的多轮按轮拆、删历史 <think>（「except for multi-step tool calls」）

详见同库笔记《Agentic SFT 的 tool-response mask 与多轮组样本》。

③轨迹数据合成（数据引擎）

agentic SFT 的成败很大程度是数据问题——高质量多轮工具轨迹稀缺。这一线的算法是「怎么自动造出可验证、可规模化的轨迹」：

工作类型算法点
AgentTuning数据多任务 agent 指令微调，混合 6 类 agent 轨迹 ＋ 通用指令，保住通用能力
Agent-FLAN数据把 agent 数据按「能力」拆解重组、识别并抑制 hallucination，提升 agent tuning 效率
xLAM / APIGen-MT数据可验证的多轮数据合成：先生成并校验「任务 blueprint」，再模拟 agent-human 交互铺成轨迹；xLAM-2 在 τ-bench/BFCL 超 GPT-4o
ToolACE数据自进化的 API 池 ＋ 多 agent 对话生成 ＋ 双层校验，造高准确度 function-calling 数据
ASTRA · Procedural Env Generation数据+RL自动合成 agentic 轨迹与「强化 arena / 程序化环境」，把数据合成与可训练环境打通

④超越均匀 CE：turn / step 级 credit

朴素 SFT 对每个 token 一视同仁，但一条长轨迹里各步贡献天差地别。这一线把「哪一步更该学/更该奖」做细——很多源自 RL 的 credit assignment，但思想正回流到 SFT/cold-start 的样本加权里：

工作类型算法点
SALTcredit用 trajectory graph 给 step 级 advantage——同任务不同轨迹「共享的步」与「分叉的步」区别对待
CriticSearchcreditretrospective critic 用全轨迹的「上帝视角」给每个 turn 稠密反馈
Beyond Trajectory Rewards（agentic search）SFT+RLgraph 建模做 step 级 credit，SFT cold-start ＋ RL 结合
InT（Self-Proposed Interventions）SFT 信号定位失败轨迹的首个错误步、自提单步 intervention、只验一步 → 稠密 step 级监督，作 RL 初始化 ＋≈10 分

⑤拒绝采样 / 自蒸馏 / 迭代 SFT（用模型自己的轨迹）

不靠外部专家，让模型自己生成轨迹 → 筛选 → 拿来 SFT，迭代自举。这是当下最活跃的一线，也在模糊 SFT 与 RL 的界限：

工作类型算法点
RFT（Rejection-sampling Fine-Tuning）/ Hint-RFT迭代SFT只留「通过 verifier / 用了工具且答对」的自生成轨迹做 SFT，多轮迭代；工具场景只收「含验证过工具调用」的样本
Exploring Expert Failures迭代SFT不只学专家成功轨迹，还挖掘专家失败里的有用片段；WebShop 胜率 35.6%→62%、可超过示范者
SDFT自蒸馏demonstration-conditioned 的自己当 teacher、on-policy reverse-KL 自蒸馏；teacher 离 base 近 → 抗 catastrophic forgetting
SSD（Simple Self-Distillation）自蒸馏按特定 temperature 采自身解、对未验证样本做 SFT，靠「分布重塑」提升 coding
Self-Distillation Zero自蒸馏self-revision 把二值 reward 变成稠密监督

⑥SFT 的病理 与 SFT↔RL 边界

越来越多工作指出：纯 cross-entropy 的 SFT 有结构性毛病——mode collapse、熵塌缩、只记忆不泛化，做后续 RL 的底子也差。于是算法往「保熵 / 用负样本 / 单阶段融合 RL」走：

工作类型算法点
SFT vs RL（post-training 研究） · Debunk SFT Generalization分析系统对比 SFT 与 RL 的泛化/记忆差异，澄清「SFT 只会记忆」的误读边界
SED-SFT · Entropy-Preserving SFTSFT对抗 mode collapse：选择性鼓励多样性 / 自适应自蒸馏保熵，给后续 RL 留探索空间
SRFTSFT+RL单阶段把 SFT 与 RL 用熵感知加权融合，+4.8 over SFT、+3.4 over SFT→RL
Learning from Mistakes（负样本） · GLOW负样本把失败/负轨迹纳入训练提升 OOD 泛化（GLOW 报 +5.51% OOD、作 RL init 抬 MMLU）

脉络与判断

三条主线 ① 监督越来越「不均匀」。 从「整段轨迹均匀 CE」走向「选择性掩码（ACC）＋ turn/step 级 credit（SALT/CriticSearch/InT）」——把梯度精准送到该学的那一步、该看的那段证据。

② 数据越来越「自给」。 从依赖人标/蒸馏大模型（AgentTuning/xLAM/ToolACE/APIGen-MT），转向用模型自己的轨迹做拒绝采样与自蒸馏（RFT/SDFT/SSD），并开始用失败样本（Exploring Expert Failures / 负样本）。

③ SFT 与 RL 在收敛。 纯 SFT 的 mode collapse / 接口捷径 / 弱泛化被反复确认，于是「保熵 SFT」「单阶段 SFT+RL（SRFT）」「SFT 作 cold-start + step-credit RL」成为主流姿势——agentic SFT 越来越不是终点，而是 RL 的起点。

对「MCP / 工具调用」从业者的实操含义

  • 数据：优先 APIGen-MT/ToolACE 式「可验证合成 + 双层校验」，并保留失败轨迹备用（拒绝采样 + 失败挖掘）。
  • 掩码：tool-response 默认 mask；但若长程证据重要，考虑 ACC 式把证据编进可学 target。
  • 多轮：带 thinking 用 One-Pass 思路省算力；纯工具多轮可整段一条。
  • 别止于 SFT：用保熵 SFT 做 cold-start，再接 step/turn 级 credit 的 RL，避免接口捷径与 mode collapse。
