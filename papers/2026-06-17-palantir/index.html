一句话 — Palantir 卖的不是 BI/数仓，而是一套「ontology（本体）+ 把数据接到真实决策与行动」的运营操作系统：政府线 Gotham、商业线 Foundry、近两年主打的 AIP 把 LLM/agent 接进企业数据与流程，靠 forward-deployed engineer（FDE）重交付打法落地。赛道整体（数据平台）已 mature，但「LLM/agent 接进企业 ontology 做可执行决策」这个新形态仍 emerging。最大威胁不是同行小厂，而是 Databricks/Snowflake + AWS/Microsoft/Google 超大厂自带 AI；最大软肋是 FDE 重交付难规模化。一个 whitespace：垂直行业的「轻量版 ontology + AIP」+ 用 AI agent 替代部分 FDE 交付，去够 Palantir 够不着的中腰部企业。

①赛道地图 + 早期度判断

Palantir 所在赛道可拆成两层。底层是「企业/政府大数据集成与运营平台」——把散落在 ERP、CRM、传感器、情报源里的异构数据汇到一处、建模、可视化、驱动操作。这层已经被大厂和成熟玩家占满（数仓/lakehouse/BI），属成熟赛道。上层是 Palantir 的差异化叙事：ontology（本体）——不只是把数据存起来查询，而是建一个「对象 + 关系 + 行动（action）」的语义层，把数据直接接到真实世界的决策与执行（派单、调度、拦截、审批），并在 2023 年后用 AIP 把 LLM/agent 嵌进这个 ontology，让模型在受控的企业上下文里「动手做事」而非只聊天。这层是新兴形态。

玩家分群（四象限别漏）

  • 现有巨头 / 直接替代：Databricks（lakehouse + Agent Bricks/Genie，估值 $134B、run-rate $5.4B 🟡）、Snowflake（AI Data Cloud，消费计费 🟡）——数据底座的最大替代，正从「数仓」向「AI 应用层」上攻。
  • 垂直创业公司 / 相邻：C3.ai（卖成品 AI 应用，FY2025 营收 $389M 🟢）、Scale AI（数据标注 + 政府/国防 AI）、Anduril（Lattice OS，抢国防预算与心智，估值 $61B 🟡）。
  • 大厂顺手做：Microsoft（Fabric / Azure + Copilot）、Google Cloud（BigQuery / Vertex / Gemini）、AWS（Bedrock / SageMaker / 数据湖全家桶）——把 AI 当云的增值件白送，是「平台被绕过」的结构性威胁。
  • 系统集成商 / 服务：Booz Allen、SAIC、Accenture、Deloitte 等——在政企交付侧既是渠道伙伴也是替代（「我们自己用现代数据栈给你搭」）。
  • 开源 / 自建（最易被忽视）：现代数据栈 dbt + Spark + Airflow + 各 BI（Looker/Power BI/Tableau） + 企业自建 LLM 应用（LangChain/向量库/自托管开源模型）——大企业「自己拼」一套能覆盖 Palantir 相当部分价值，且无 license 锁定。
早期度判断 mature（底层数据平台）  +  emerging（LLM/agent 接 ontology 做可执行决策）

判断信号：① 玩家众多且巨头全部进场（AWS/MS/Google + Databricks/Snowflake），数据集成/BI 这层早已红海 → 成熟；② 但「把 LLM agent 接进企业语义层、让它在受控上下文里执行操作」尚无公认 benchmark/标准、客户认知刚成型、各家叙事打架（ontology vs lakehouse vs copilot）→ 新兴；③ 融资/资本热度极高（Databricks 一年内 Series K→L、估值 $100B→$134B 🟡；Anduril $61B 🟡），说明资本认为「AI×企业数据×决策/国防」仍有大空间；④ Palantir 自身 US 商业 +133% YoY 🟢 的爆发，是「AIP 拐点」的强信号——需求侧刚被 LLM 点燃。

读矩阵：Palantir 的真差异化在两格——「ontology 把数据接到可执行行动」与「FDE 重交付」，这让它在「数据很乱、决策要落地、容错低」的政府/国防/大型工业场景里很强，竞品大多停在「存好查好、给个 copilot」。但它的软肋同样写在矩阵里：交付模式那一列只有它（和集成商）是重人力——难规模化、毛利与扩张速度受限；而数据/壁垒那一列，超大厂的「云锁定 + 默认分发」是它绕不过的结构性压制。同质化最严重的是底层数据集成（人人能做），最不同质化的是「LLM agent × 企业语义层 × 受控执行」这层——也正是它当前增长的来源。

⑥中国对标（政企大数据决策 / AI 中台）

中国做「政企大数据决策 / AI 中台」的玩家与 Palantir 形似神不同。共性是「把政企的数据汇起来 + 上层做分析/决策/AI 应用 + 重交付」；差异在政企关系、国产化/信创要求、以及更偏「项目制集成」而非「自有平台高毛利订阅」。

「Palantir 模式」在中国的落地形态判断：① 更偏交付制、更碎片化——政企采购重关系、重定制、重信创，难复制 Palantir「自有平台 + 高毛利订阅」的飞轮，多数玩家毛利与盈利性弱于 Palantir（4Paradigm 仍亏损但在收窄 🟡）；② 国产化/信创是双刃——给了本土玩家护城河（外资平台进不来），但也把市场切碎、价格战、回款慢；③ ontology「接决策/行动」这层最薄——多数停在数据汇聚 + BI + 图谱，真正「LLM agent 在受控企业上下文里执行操作」的标杆产品尚未出现。whitespace（中国）：做一个真正「轻量 ontology + AIP + 信创合规」的垂直行业 agent 产品，把项目制交付用 AI 降本、再产品化复制，是 Palantir 中国对标里仍空着的格子。

⑦创业 idea 借鉴

Whitespace / 机会 垂直行业的「ontology + AIP」轻量版 + 用 AI agent 替代部分 FDE 交付。 Palantir 强在「重交付的可执行平台」，但 FDE 太贵、起步合同太大，把中腰部企业（够不着 Palantir 报价/交付门槛的客户）整片留空。在某个垂直行业（如医疗供应链、制造排产、保险理赔、能源调度）把「对象+关系+action」的 ontology 做成预置模板，再用 LLM agent 自动完成大半数据接入与建模（即 Palantir 自己也在做的 AI FDE 方向），用更轻的交付去够这片客户。

现有玩家共同短板（unmet needs）

① 落地太贵太慢：Palantir 靠 FDE 重交付，起步成本与周期把中小客户挡在门外；集成商更贵且无平台复用。② 大厂 copilot/数仓「只存好查好、不接行动」：从「看到洞察」到「真在系统里执行操作」之间有断层，缺一个受控、可审计的「数据→决策→行动」闭环。③ 开源自建栈门槛高：需要强工程团队，多数行业客户拼不出来也维护不动。④ 政企/合规场景的可信 agent 缺标准：谁来保证 LLM 在敏感数据上「只做被授权的事」。

Why now（为什么现在值得做）

LLM 让「把自然语言意图翻译成对数据/系统的受控操作」第一次变得可行——这正是把「数据接决策」的门槛大幅拉低的技术拐点。Palantir 自己的 US 商业 +133% YoY 与 2026/03 推出 AI FDE agent，都是「需求侧刚被点燃、供给侧刚能做」的双重信号。AI 也让重交付这件事本身可以被 agent 降本——过去必须驻场建模的活，现在可由 agent 完成相当一部分，使「Palantir 式深度」与「SaaS 式轻交付」第一次有机会兼得。

可借鉴的打法 + 相邻机会

  • borrowable：(a) ontology 优先——先把客户业务建成「对象+关系+action」语义层，再让 AI 在上面跑，比直接「数据 + chatbot」黏性高得多；(b) 从一个高容错敏感的垂直场景切入（Palantir 从情报界起家），在那里把可信/可审计的执行闭环打磨成标杆，再横向复制；(c) 先重交付建信任、再产品化降本的两段式 GTM。
  • adjacent：(a) 给现代数据栈/Databricks 上面做一层「ontology + agent 执行」中间件（不跟底座竞争、寄生其上）；(b) 专做「AI FDE」工具——把驻场工程师的活做成 agent 产品，卖给所有想做 Palantir 式落地但请不起 FDE 的公司/集成商；(c) 行业垂直版「决策 OS」（医疗/制造/保险/能源各一），把合规与行业 ontology 预置好。
风险（诚实给） ① 巨头碾压：AWS/Microsoft/Google 把 AI 当云增值件白送，Databricks/Snowflake 从数据底座上攻，轻量玩家随时被「顺手做掉」。② 重交付难扩——你想用 AI 降本，但 Palantir 也在做 AI FDE，可能用规模与数据反向碾压你的轻量切口。③ 政企销售周期长、回款慢、重关系/资质，烧钱久；中国还叠加信创与价格战。④ 数据合规与安全：在敏感数据上跑 agent，一次事故就可能终结信任。⑤ 「可执行决策」尚无公认 benchmark/标准——既是机会也意味着客户教育成本高、ROI 难证明。

⑧现实判断

Palantir 当下的胜算很实：它把「ontology + 接到真实决策/行动 + FDE 重交付」做成了竞品难一夜复制的组合，政府/国防的多年期合同 + 美国商业 AIP 爆发让它同时高增长又 GAAP 盈利（Rule of 40 = 145%），这是大多数对手做不到的。它真正的软肋有三条：(1) FDE 重交付难规模化——增长越快、人力越紧，毛利与扩张受制（它自己在用 AI FDE 解，但这恰好也会被创业者借用）；(2) 超大厂的结构性压制——云锁定 + 默认分发让「平台被绕过」的风险长期存在，Databricks/Snowflake 又从底座上攻；(3) 估值与叙事的预期透支——市值 ~$308B、P/S 极高（2026 年内股价一度回撤 🟡），任何增速放缓都会被放大。对创业者，最现实的读法是：不要正面打平台，去占它够不着的「中腰部 + 垂直 + 轻交付」格子，用 AI agent 把交付成本打下来——那是矩阵里目前空着、且 why-now 成立的一格。
