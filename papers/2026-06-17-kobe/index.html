一句话 — Kobe（getkobe.ai）是一个替个人「读」自己工作流的 AI career agent：你授权它接入 Slack / Notion / Granola 等工具，它自动从你真实的工作（你驱动的项目、做的决策、解决的问题）里识别并量化技能，生成带「同行 percentile」的技能画像，并在你主动 opt-in 招聘 marketplace 后被动地把你匹配给公司——卖点是「raw data 只存本地、不上服务器」。赛道判 emerging（局部 nascent）：「主动读工作流→量化技能」这个切入点目前几乎没人正面做（Mercor / Micro1 是 B 端专家供给、Teal / Jobright 是简历/投递 copilot），whitespace 真实存在；但 Kobe 本身信息极少、官网当前甚至返回部署错误，产品实效完全未验证，双边冷启动与隐私信任是死穴。

①赛道地图 + 早期度判断

Kobe 处在三条线的交叉口：(a) 个人工作数据采集（像 Granola/Notion 那样从你的工具里抽取工作痕迹）、(b) 技能量化 / work intelligence（把工作痕迹转成「可验证的技能画像 + 同行 percentile」）、(c) 招聘双边匹配（把这份画像变成被动求职、对接公司）。它要解决的核心需求是：简历是自述、滞后、可注水的；而「你到底会什么」其实藏在你每天的真实工作里——Kobe 想用 agent 直接读这些工作、产出「proof-based」的技能与匹配。🟡

玩家分群（四象限别漏）

  • 现有巨头：LinkedIn（职业身份 + 招聘的事实标准，已大量铺 AI；但其数据仍是「用户自填 + 社交」而非「读工作流」）；Microsoft（Viva / Copilot 已经在企业内读你的 Teams/邮件/文档做 work intelligence，是离 Kobe「读工作」最近的巨头，只是它服务雇主、不服务个人求职）。🟡
  • 垂直创业公司：Mercor（$10B，B 端专家人才/AI 训练数据）、Micro1（$500M，B 端 AI 招聘+数据）、Teal、Jobright、Careerflow、Simplify、Cord、Juicebox（招聘方搜人）——大多做简历优化 / 自动投递 / 招聘方找人，没有一个正面做「agent 主动读你工作流量化技能」。🟡
  • 大厂顺手做：Microsoft Viva Skills / LinkedIn Skills、Google、Notion AI、Slack AI 都握着「你的工作数据」，理论上离 Kobe 一步之遥——这是 Kobe 最大的结构性威胁（数据在人家手里）。🔴 推断
  • 开源 / 免费替代：暂无「读工作流量化技能」的开源标准品；但「AI 简历 / 自动投递」侧有大量免费/freemium（Teal 免费档、各 job copilot 浏览器插件），会从下方挤压 Kobe 的「匹配」价值。🟡
早期度判断 emerging（「主动读工作流量化技能」这一格更接近 nascent） （nascent 萌芽 / emerging 新兴 / crowded 拥挤 / mature 成熟）

判断信号：① 大赛道「AI + 招聘 / career」整体已经 crowded（Mercor $10B、Micro1 $100M ARR、一堆 job copilot），资本与玩家密集；② 但 Kobe 真正占的细分——「个人 agent 主动读真实工作流 → 量化技能 → 被动匹配」——目前几乎没有正面玩家，更接近 nascent；③ 巨头（LinkedIn / Microsoft）握着数据但尚未把它做成「个人侧、跨雇主、proof-based」的产品；④ 无公认 benchmark / 标准（「技能 percentile」如何度量、如何被雇主信任，都还没成型）；⑤ Kobe 自身处于 waitlist / 早期访问阶段、官网部署都不稳定，产品实效未验证。综合判 emerging，其切入点局部 nascent。

⑥中国对标（这套「个人工作 agent + 招聘双边」在中国可行吗）

把 Kobe 的三块能力拆开看中国玩家：

  • 招聘双边平台：BOSS 直聘已大量铺 AI——自研「南北阁」大模型 + 接入 DeepSeek，招聘端有帮 HR 搜人/匹配的 AI Agent（自述沟通效率 +25%），求职端有 AI 搜索、求职助手、AI 模拟面试、自动生成「牛人优势」简历优化。🟡 它是中国「双边网络」事实标准，等于 Kobe 要面对的「中国版 LinkedIn+Indeed」。
  • 中高端职业社交 / 人脉：脉脉（Maimai）——可投递、可直接搜企业员工建人脉聊机会；自报 2026 年 1–2 月 AI 岗位数同比涨约 12 倍、AI 岗均月薪 ¥60,738。🟡 最接近「LinkedIn 式个人职业身份」，但数据仍是自填+社交，不读工作流。
  • AI 面试 / 测评：近屿智能（AI 得贤招聘官）等做 AI 面试/能力测评——对应 Micro1 的 Zara 那一层，偏 B 端雇主侧。🔴 未在本次检索中独立核到细节
  • 个人工作数据 / RPA agent：实在智能等 RPA / agent 玩家握「读软件操作」的能力，但目标是企业自动化，没人把它转成「个人跨雇主技能画像 + 求职」。🔴 推断
  • 个人简历 AI：大量免费/低价简历优化、自动投递工具，红海。
中国可行性 + whitespace 「个人 agent 主动读工作流 → 量化技能」这一格，中国同样基本空着，whitespace 成立。但落地有三道强阻力：① 数据合规——中国《个人信息保护法》《数据安全法》对「读员工的工作数据」极敏感，本地存储反而可能成为中国市场的核心卖点甚至合规前提；② 工具生态不同——要接的是钉钉/飞书/企业微信/Notion 中国替代品，且企业 IT 管控强，个人未必有权导出工作数据；③ 双边被 BOSS 直聘/脉脉锁死，冷启动比美国更难。更现实的中国打法：不做独立双边平台，而是做「个人侧的飞书/钉钉技能 agent」（帮你把工作沉淀成可携带的技能档案，对接现有招聘平台），把「读工作流量化技能」当能力层卖，而非自建 marketplace。

⑦创业 idea 借鉴

Whitespace / 机会 定位图右上象限——「主动读真实工作流 × 服务个人」——目前几乎没人正面占。简历是自述、滞后、可注水；而「你真正会什么」藏在你每天的 Slack/文档/会议里。谁能把这层「proof-based、跨雇主、个人拥有」的技能画像做成被雇主信任的标准，谁就握住了「下一代简历」。这正是 Kobe 押的注，也是它最值得借鉴的一点。

现有玩家共同短板（unmet needs）

① 简历/LinkedIn 是自填，雇主天然不信；② Mercor/Micro1 把价值收在 B 端，个人拿不到自己的「可携带技能资产」；③ job copilot 们只优化「投递动作」，不改变「凭据是自述的」这个根；④ 巨头（LinkedIn/Microsoft）虽握工作数据，但要么是自填社交、要么锁在雇主租户里、不给个人跨雇主使用。共同缺口 = 一个个人拥有、读真实工作、被雇主信任的技能凭据层。

Why now（为什么现在值得做）

① agent 成熟——LLM 现在能真的「读懂」非结构化工作内容（聊天/会议/文档）并抽象成技能，三年前做不到；② work data 上云且可接——Slack/Notion/Granola 等都有 API，工作痕迹首次可被程序化读取；③ AI 人才数据需求井喷——Mercor/Micro1 证明「精准的人才/技能数据」值大钱；④ 隐私焦虑 + 本地推理——端侧模型让「raw data 不出本地」从口号变得技术可行，正好对冲「读我工作」的最大顾虑。

可借鉴的打法 + 相邻机会

  • borrowable：（a）「读工作流而非读简历」这个数据切入点，可迁到任何需要「证明能力」的场景（freelancer 接单、晋升评审、内部转岗、保险/信贷的职业画像）；（b）「本地存储」当信任卖点——隐私即 GTM，可复制到任何要碰用户敏感数据的 agent；（c）「被动求职」——画像常驻、机会找上门，而非用户主动投递。
  • adjacent：（a）与 Mercor 互补而非竞争——Kobe 做「个人供给侧画像」，把高质量、proof-based 的人才供给喂给 Mercor/Micro1 这类 B 端 marketplace，做「人才数据的上游」；（b）企业内「技能图谱」——同一引擎对内卖给雇主做内部人才盘点/晋升（即 Microsoft Viva 的个人友好版）；（c）面向 AI 训练的「专家发现」——读工作流天然能识别谁是某领域真专家，正是 AI 数据标注最缺的。
风险（诚实给） ① 双边冷启动——没有求职者就没有雇主，反之亦然，这是招聘平台的经典死亡谷，Kobe 还要先攒到足够画像才有 marketplace；② 隐私信任——让 agent 读你全部工作是巨大的信任门槛，「本地存储」是必要但未必充分的承诺（🔴 自述，未独立验证）；③ 巨头碾压——LinkedIn 一旦做「verified skills from work」、或 Microsoft 把 Viva 个人化，Kobe 的数据与分发都被降维打击；④ 与 Mercor 同质化的诱惑——若被迫往 B 端供给走，就直接撞上 $10B 的 Mercor；⑤ 「percentile」可信度——雇主凭什么信 Kobe 的技能分？没有公认标准前，这套凭据的价值悬空；⑥ 极早期执行风险——团队/融资不明、官网部署都不稳，产品能否做出来本身存疑。

⑧现实判断

Kobe 的叙事很对、卡位很巧：在一个已经 crowded 的「AI×招聘」大赛道里，它找了一个别人没正面做的 nascent 小格子（个人 agent 读工作流量化技能），且用「本地存储」直接拆掉这个切入点最大的隐私炸弹。如果「读工作→proof-based 技能凭据」真能被雇主接受，它就是「下一代 LinkedIn 资料」，天花板很高，并且能和 Mercor/Micro1 的 B 端形成「供给侧上游」的互补而非正面竞争。

但软肋同样致命且现在就看得见：双边冷启动、隐私信任、巨头随时降维、技能 percentile 缺公认标准——任何一条都可能卡死。叠加团队/融资/用户全部未披露、官网当前抓取都失败，能核实的只有「方向」，产品实效完全未验证。结论：作为idea 来源，「读工作流的个人技能 agent + 隐私本地化 + 与 B 端 marketplace 互补」非常值得借鉴；作为一家公司，现在只能判为高赔率、高不确定的极早期下注，一切待其披露团队与真实产品后再校准。
